# Swagger и качество API-документации

Документ фиксирует текущее состояние API-документации проекта «Рацион» и критерии, по которым она считается готовой для демонстрации, тестирования и защиты курсовой работы.

## Точки входа

- Swagger UI: `GET /api/docs`
- OpenAPI JSON: `GET /api/openapi.json`
- Health check: `GET /api/health`

Swagger UI подключен к тому же OpenAPI-документу, который отдает сервер. Это значит, что ручная проверка через браузер и автоматические тесты смотрят на один источник правды.

## Что покрывает OpenAPI

- Авторизация: регистрация, вход, текущий профиль и JWT Bearer auth.
- Питание: цели КБЖУ, продукты, приемы пищи, шаблоны, рецепты и план питания.
- Аналитика: dashboard, вода, самочувствие, замеры тела, заметка дня и избранное.
- Обмен данными: импорт `JSON/CSV/TSV`, экспорт дневного отчета в `JSON/CSV`, клиентский экспорт в `PDF`.
- Роли: публичные маршруты, маршруты пользователя и административные операции с каталогом продуктов.

## Правила оформления

- У каждого маршрута есть `tag`, `summary`, `operationId` и успешный `2xx`-ответ.
- У защищенных маршрутов явно указан `bearerAuth` и ответ `401`.
- У маршрутов с path-параметрами описаны сами параметры.
- У типовых ошибок есть общие схемы: `ValidationError`, `UnauthorizedError`, `ForbiddenError`, `NotFoundError`, `ServerError`.
- У ключевых доменных сущностей есть схемы: пользователь, цели, продукт, прием пищи, вода, шаблон, рецепт, check-in, замер тела, план питания, покупка, заметка дня и избранное.

## Визуальная настройка Swagger UI

Swagger UI оформлен в спокойной светлой стилистике проекта:

- брендированный заголовок `Рацион API`;
- закрепленная верхняя панель;
- включенный `Try it out`;
- включенный фильтр по маршрутам;
- сохранение авторизации между запросами;
- отображение длительности запросов;
- компактное отображение path и HTTP-методов.

## Как проверить вручную

1. Запустить проект через `docker compose up --build`.
2. Открыть `http://localhost:8080/api/docs`.
3. Выполнить `POST /api/auth/login` с `demo@nutritrack.local / Demo123!`.
4. Нажать `Authorize` и вставить JWT.
5. Проверить защищенные маршруты: `GET /api/dashboard`, `GET /api/meals`, `GET /api/exports/daily-report`.
6. Для роли администратора войти как `admin@nutritrack.local / Admin123!` и проверить `POST /api/products`.

## Автоматические проверки

Качество Swagger и документации проверяется тестом:

```bash
npm test
```

Отдельный файл тестов: `server/tests/openapi-docs.test.js`.

Проверяются:

- читаемость русских строк в OpenAPI;
- наличие схем, тегов, operationId и ответов;
- явное описание защищенных маршрутов;
- наличие path-параметров;
- настройки Swagger UI;
- ссылки между README, docs и wiki.
